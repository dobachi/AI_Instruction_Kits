## コンサルテーションフレームワーク

{{#if consultation_framework}}
### 採用フレームワーク: {{consultation_framework}}

{{#if (eq consultation_framework "構造化問題解決")}}
**マッキンゼー式問題解決アプローチ**
1. **問題の定義**
   - 真の問題の特定
   - 問題の構造化と分解
   - 優先順位の設定

2. **仮説の設定**
   - イシューツリーの作成
   - 検証可能な仮説の構築
   - クリティカルファクターの特定

3. **分析の実行**
   - データ収集と分析
   - 事実とインサイトの分離
   - 仮説の検証・修正

4. **解決策の策定**
   - オプションの生成
   - 評価基準の設定
   - 最適解の選択

5. **実装計画**
   - アクションプランの策定
   - リソース配分
   - リスク管理
{{/if}}

{{#if (eq consultation_framework "デザイン思考")}}
**人間中心設計アプローチ**
1. **共感（Empathize）**
   - ステークホルダーの理解
   - ペルソナの作成
   - ジャーニーマップの作成

2. **定義（Define）**
   - 問題の再定義
   - ニーズの明確化
   - 機会領域の特定

3. **創造（Ideate）**
   - ブレインストーミング
   - 創造的解決策の生成
   - アイデアの収束

4. **プロトタイプ（Prototype）**
   - 概念実証の作成
   - 早期検証
   - 反復改善

5. **テスト（Test）**
   - ユーザーテスト
   - フィードバック収集
   - 解決策の改良
{{/if}}

{{#if (eq consultation_framework "リーン手法")}}
**リーンスタートアップ手法**
1. **仮説構築**
   - ビジネスモデル仮説
   - 顧客仮説
   - 価値仮説

2. **MVP開発**
   - 最小限の実行可能な製品
   - 学習目標の設定
   - 測定可能な指標

3. **測定**
   - データ収集
   - 学習の検証
   - 仮説の評価

4. **学習**
   - インサイトの抽出
   - 方向転換の判断
   - 次の仮説設定

5. **反復**
   - 継続的改善
   - スケールアップ
   - ピボット判断

{{#if (eq consultation_framework "アジャイル")}}
**アジャイル手法**

   - スプリント計画
   - 段階的価値提供

3. **適応的計画**
