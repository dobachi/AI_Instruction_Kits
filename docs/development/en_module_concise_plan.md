# 英語版モジュール簡潔化実施計画

## 目的
英語版モジュールに簡潔版を作成し、日本語版と同じ命名規則を適用する

## 現状
- 総モジュール数: 64
- すべて詳細版として存在（.md）
- _concise.mdや_detailed.mdは存在しない

## 命名規則（日本語版と同じ）
- `module.md` - 簡潔版（デフォルト）
- `module_detailed.md` - 詳細版

## 実施手順

### Phase 1: 準備
1. 作業用ディレクトリの作成
2. バックアップの作成
3. 変換スクリプトの準備

### Phase 2: 簡潔版の作成
1. 日本語版のPhase 4.5ガイドラインに基づいて作成
2. カテゴリごとに並列処理で実施
3. 目標サイズ: 元のサイズの20-30%

### Phase 3: ファイル名の変更
1. 現在の.mdファイルを_detailed.mdにリネーム
2. 新しく作成した簡潔版を.mdとして保存

### Phase 4: 検証
1. ファイル数の確認
2. 命名規則の確認
3. composer.pyでの動作確認

## リスク管理
- 各フェーズでバックアップを作成
- 段階的に実施し、各段階で確認
- エラー発生時は即座に停止

## 成功基準
1. 64個の英語版モジュールすべてに簡潔版と詳細版が存在
2. 命名規則が正しく適用されている
3. composer.pyで正しく動作する
